# LeClaw Security

LeClaw is open source. Read the code before you trust it.

---

## What LeClaw Reads

LeClaw requests the minimum permissions required for each agent:

| Agent | HubSpot Scopes | Salesforce Objects |
|-------|---------------|-------------------|
| Le Data Quality | `crm.objects.contacts.read` `crm.objects.companies.read` | Contact, Account (read only) |

We never request write permissions unless you explicitly enable write-back mode.

---

## What LeClaw Stores

**Shadow mode (default — free tier):**
- Nothing. The agent reads, analyzes, and discards.
- Only the run log is stored: issue counts, severity levels, timestamps.
- Raw CRM records are never persisted.

**Write-back mode (paid tier):**
- An audit log of every change made: what field was changed, what value, when, by which agent.
- Raw records are still never stored — only change deltas.

---

## What Gets Sent to Claude API

The AI summary is generated by sending metadata to Claude — never raw CRM records.

What gets sent:
```
Total contacts: 247
High severity issues: 3
Medium severity issues: 12
Top issues: ["Contact missing email", "Company missing domain", ...]
```

What never gets sent:
- Contact names, emails, phone numbers
- Company names, domains, revenue figures
- Deal names, amounts, close dates
- Any PII or sensitive business data

---

## Credential Storage

Your CRM tokens are stored in `.env` on your local machine (self-hosted) or in Supabase Vault with AES-256 encryption (hosted product). Tokens are never logged, never exposed in API responses, never sent to third parties.

---

## Revoking Access

**HubSpot:** Settings → Integrations → Private Apps → Delete app
**Salesforce:** Setup → Connected Apps → Revoke

Revoking immediately terminates all LeClaw access. No action required on the LeClaw side.

---

## Self-Hosted Security

Running LeClaw self-hosted means:
- Your CRM tokens never leave your infrastructure
- No data passes through LeClaw servers
- The only external call is to Claude API (metadata only, as described above)
- You control the schedule, the permissions, and the data
